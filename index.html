🤖 IA & Agentes

IA · LLMs Avançado

Rede Transformer →

Do texto cru à próxima palavra: tokenização, embeddings, positional encoding, self-attention (Q/K/V), multi-head, residual, feed-forward e cabeça de saída.

 IA · RAG Intermediário

RAG →

Retrieval-Augmented Generation: chunking, embeddings, busca vetorial, recuperação de contexto e como fundamentar as respostas de um LLM em dados reais.

 IA · Protocolos Intermediário

Model Context Protocol →

Como o MCP conecta LLMs a ferramentas e dados externos: Host, Client, Server, Tools, Resources, Prompts e o fluxo JSON-RPC de ida e volta.

 IA · Agentes Intermediário

SubAgentes →

Orquestrador que spawna subagentes especializados, executa em paralelo, agrega resultados e lida com falhas — o padrão multi-agente explicado.

 IA · Engenharia Intermediário

Engenharia de Contexto →

Janela de contexto, hierarquia de mensagens, estratégias de compressão, caching de prompt e como aproveitar cada token da forma mais eficiente.

 IA · Prompts Iniciante

Engenharia de Prompt →

Zero-shot, few-shot, chain-of-thought, role prompting, ReAct, XML tags e os anti-padrões que fazem modelos perderem precisão.

 IA · Segurança Intermediário

Guardrails →

Camadas de proteção para sistemas de IA: validação de entrada, detecção de injeção de prompt, filtros de saída, fallback e auditoria.

 IA · Agentes Intermediário

Skills →

O que são Skills no contexto de agentes, como o LLM decide qual invocar, o contrato input/output e como compor skills em fluxos mais complexos.

 IA · Metodologia Intermediário

Spec-Driven Development →

O ciclo SDD: escrever a spec, gerar código com IA, validar com testes automatizados, iterar — e por que a spec é a fonte de verdade.

 IA · Especificações Intermediário

Specs para Agentes →

Estrutura de uma spec de agente: contexto, objetivo, restrições, tools, contratos de I/O, exemplos e como diferenciar de system prompt e CLAUDE.md.

🌐 Web & Protocolos

Web · Redes Iniciante

Requisição HTTP →

Da URL digitada à página renderizada: DNS, conexão TCP, handshake TLS, request, resposta (200 OK) e cache — em diagrama de sequência.

 Redes Iniciante

TCP/IP →

Three-way handshake (SYN/SYN-ACK/ACK), segmentos numerados, ACKs, janela deslizante, perda + retransmissão e encerramento.

 Protocolos · Pagamentos Avançado

ISO 8583 →

O padrão de mensagens de cartão: MTI, bitmap primário/secundário, data elements e o fluxo POS → Adquirente → Bandeira → Emissor.

 Web · Real-time Intermediário

WebSocket →

Full-duplex sobre TCP: handshake HTTP 101 Upgrade, estrutura de frame (FIN, opcode, MASK), opcodes text/binary/ping/pong e comparativo com SSE.

 Web · APIs Intermediário

gRPC →

Schema-first com Protocol Buffers, HTTP/2 multiplexing, quatro tipos de RPC (Unary, Server Stream, Client Stream, Bidirectional) e Interceptors.

 Web · Segurança Avançado

TLS / mTLS →

Handshake ClientHello/ServerHello, X.509 e chain of trust, ECDHE key exchange, derivação de session keys e mTLS para autenticação mútua.

🔐 Segurança & Autenticação

Auth · OAuth Avançado

OAuth 2.0 / OIDC →

Authorization Code Flow com 4 atores, PKCE para apps públicas, os três tokens (access, refresh, id_token) e OIDC como camada de identidade.

 Auth · Tokens Intermediário

JWT →

Header.Payload.Signature: claims padrão (sub, exp, iss, aud), HS256 vs RS256, verificação stateless sem banco e estratégia de revogação com refresh token.

 Auth · SSO Avançado

SSO — Single Sign-On →

1 login no IdP, acesso a N apps: SP-initiated flow, SSO session cookie, assertions SAML vs OIDC, e Single Logout (SLO).

 Auth · Sessions Iniciante

Sessions & Cookies →

State server-side com session_id no Redis, atributos HttpOnly/Secure/SameSite, proteção contra CSRF e session hijacking.

 Auth · M2M Iniciante

API Keys →

Geração com crypto.randomBytes, armazenar apenas SHA-256, rate limiting por chave no Redis, escopos e rotação sem downtime.

 Auth · MFA Intermediário

MFA / 2FA →

Os 3 fatores (saber, ter, ser): TOTP com HMAC-SHA1 e janela de 30s, FIDO2/WebAuthn phishing-resistant e por que SMS OTP é inseguro.

 Segurança · Crypto Avançado

Criptografia →

Simétrica (AES, ChaCha20) vs assimétrica (RSA, ECC), hybrid encryption, assinaturas digitais, funções hash e KDFs (bcrypt, Argon2).

⛓️ Sistemas Distribuídos

Distribuídos · Particionamento Avançado

Consistent Hashing →

Hash ring 0..2³², lookup clockwise O(log N), virtual nodes para distribuição uniforme, add/remove nó remapeia apenas ~1/N chaves. Base de Cassandra, DynamoDB e Redis Cluster.

 Distribuídos · Consensus Avançado

Blockchain →

Cadeia de blocos encadeados por prev_hash, Proof of Work (brute-force de nonce), longest chain wins, ciclo de vida de uma transação e Smart Contracts.

 Kubernetes · Redes Intermediário

Ingress →

 Resiliência Intermediário

Circuit Breaker →

 Resiliência Intermediário

Rate Limiting →

Token Bucket, Leaky Bucket e Sliding Window: como proteger APIs de abuso, tokens se reabastecendo, burst capacity e os headers X-RateLimit-*.

 Infra · Redes Iniciante

Load Balancer →

L4 vs L7, Round Robin, Weighted, Least Connections, IP Hash, health checks, SSL termination e Global Load Balancing.
